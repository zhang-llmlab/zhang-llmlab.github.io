## 工具层是 Agent 的“刹车系统”

很多 Agent 不稳定，并不是模型差，而是工具层没有治理好。

一个可上线的工具调用层至少要包含：

- 参数校验（Schema）
- 调用超时（Timeout）
- 重试策略（Retry + Backoff）
- 降级策略（Fallback）
- 可观测性（日志、trace、错误码）

## 推荐的执行策略

### 1) 明确工具契约

每个工具都要有清晰输入输出，不允许“模糊字符串协议”。

### 2) 失败分级

- 可重试失败：超时、限流、短暂网络错误
- 不可重试失败：参数错误、权限不足、业务校验失败

### 3) 重试不是无限循环

设置最大尝试次数和整体预算，例如：

- 单工具最多 2 次重试
- 全任务工具调用总时长不超过 20 秒

## 示例伪代码

```ts
async function callToolWithGuard(tool, input) {
  for (let i = 0; i < 3; i++) {
    try {
      return await withTimeout(tool(input), 8000);
    } catch (e) {
      if (!isRetryable(e) || i === 2) throw e;
      await sleep(300 * (i + 1));
    }
  }
}
```

## 小结

Agent 的上限由模型决定，下限由工程治理决定。  
工具层治理做得好，系统稳定性会有质变。
