## 为什么 Agent 需要单独评估

Agent 不是单次生成，它是“多步决策 + 工具执行”的系统。  
只看最终答案对不对，会漏掉大量风险。

## 四类核心指标

### 1. 效果指标

- 任务成功率
- 关键步骤正确率
- 人工回退率

### 2. 效率指标

- 平均响应时延
- P95/P99 延迟
- 单任务 token 成本

### 3. 稳定性指标

- 工具调用失败率
- 重试次数分布
- 超时率

### 4. 安全与合规指标

- 越权调用拦截率
- 敏感信息泄露率
- Prompt 注入防护通过率

## 上线前最小验收清单

- [ ] 至少 50 条代表性任务集回放通过
- [ ] 高风险场景有兜底策略（人工接管/降级）
- [ ] 关键链路可观测（trace + tool logs）
- [ ] 成本阈值与报警已配置
- [ ] 失败样本可复盘

## 小结

评估不是“做个分数”，而是为上线建立可解释的信任机制。  
当你能回答“哪里会错、错了怎么办”，Agent 才真正可用。
