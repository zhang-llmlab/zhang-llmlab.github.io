> 读前提示：不必先懂信息检索的倒排索引细节，但最好用过某家 embedding API 或开源句向量模型。

## 一、嵌入（embedding）在做什么

把文本（词、句、段）映射到**固定维度的稠密向量**，使得语义相近的文本在空间中彼此靠近。这是语义搜索、聚类、RAG 里「先召回再精读」的基础。

## 二、相似度怎么度量

常见做法是对向量做 L2 归一化后算**余弦相似度**，或等价地在归一化后使用**点积**。工程上要统一：维度、是否归一化、距离定义（cosine / L2 / inner product）要与向量库索引配置一致。

## 三、和关键词检索的互补

- **关键词（BM25 等）**：字面匹配强，专有名词、型号、错误拼写有时更稳。
- **向量检索**：同义改写、跨说法、语义泛化更好。

混合检索（hybrid）在问答、客服、内部文档场景中往往比单一通道更稳。

## 四、RAG 里的一处常见误区

「嵌入相似」≠「事实正确」。检索只负责**候选集合**；生成阶段仍需要引用约束、重排序（rerank）、以及评测（命中、忠实度）。

## 五、小结

嵌入把文本变成可检索的几何对象；语义搜索的核心是**同一向量空间里的近邻问题**。上线前务必用业务语料做小集评测，再调模型与阈值。
